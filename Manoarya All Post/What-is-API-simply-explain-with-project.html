What is API and How it Works ?

Assume we are at a hotel and I order something and here we have our order.

Let's see this situation carefully. First we went to the hotel and sat on the table and we saw the menu of the hotel.

After seeing this menu called the waiter, they came here.

Now we told him about our order. Then the waiter goes to the kitchen and requests the items given by this order.

Now the waiter came to me with our order and now we eat whatever I ordered.

In this situation, me as an application or website and kitchen is a system of any company.

And I want some data from the system. Just like we want my order at the hotel.

So we can't go directly to the kitchen just like our Apps or websites can't go directly to the system.

Now what do we do? Yes , we told the waiter to complete this order. Just like that we told an API to respond to our request.

If we order or request something two situations come up.

First, if our order exists, that gives me the ordered items. And if my order did not exist then the waiter called me sorry.

Just like that API works if our requested data exists then responds to our results. And If data does not exist then respond to me as requests failed.

I hope you understand the concept of the API.

What does the API stand for?

API Stand for Application Programming Interface.

Which is work like a waiter or we say like a massager to interact between apps and systems to request and response data.

Types of API.

The API mainly four types of

1. Open API
2. Partner API
3. Internal API
4. Composite API

Let's learn about four kinds of APIs

1. Open API : Open API is an API to use publicly that's why we called it

public api. Means everywhere in this world we can use it without any restrictions.

For example : Oauth API by Google.

2. Partner API : Partner API is an API which we can't use without license.

3. Internal API : Internal API is an API built by a company to boost their own productivity and batter team work.

It is used privately or inside of a company.

For Example : US today and guidance have own Internal API to boost their productivity

4. Composite API : Composite API is an API to combine different data APIs and services APIs.

For Example : We add a product to cart API and for creating users bills and other information to use other APIs.

4. How Do API Works

We know API works between two systems. So we have a Clint and server.

Clint means our Apps or websites where we want to show data that we fetch from API.

Server means a storage where a data store and API goes into that storage and finds our request result.

So we have four different ways to communicate with clients and servers.

1. SOAP API
2. RPC API
3. WebSocket API
4. REST API

1. SOAP API : SOAP stands for Simple Object Access Protocol.

SOAP communicates between servers and clients using XML.

SOPE was more popular in the past but now we have more standard and advanced services to communicate with clients and servers.

2. RPC stands for Remote Procedure Cells.

RPC communicates between servers and clients using a function.

Clients execute a function on the server and the server sends an output to the client.

3. WebSocket API : WebSocket uses JSON Object to pass data.

WebSocket supports two way communications between clients and servers.

The WebSocket API server sends a callback message to the connected clients.

4. REST Stands for Representational State Transfer.

In REST API clients used GET, PUT, DELETE.etc functions to access servers data.

In REST API clients and servers communicate using HTTP(Hypertext Transfer Protocol).

The REST API works on statelessness. means servers can't store or save data when any client sends requests.
