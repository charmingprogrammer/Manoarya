    A Basic Git Cheat Sheet for you to starting verson control.

    Category : Explanation

    Git is a popular version control system that is widely used by developers for managing and tracking changes to their codebase. It allows teams to work on code simultaneously, and helps to prevent conflicts when multiple people are making changes to the same files.

    To help you get started with Git, here is a cheat sheet of some common Git commands:

    git init: This command is used to initialize a new Git repository. It creates a hidden .git folder in the current directory, which is used to store all of the repository's metadata.

    git clone <repository> : This command is used to create a local copy of a remote Git repository. It allows you to download all of the files and history of the repository to your local machine.

    git add <file>: This command is used to stage a file for commit. It tells Git that you want to include the file in the next commit.

    git commit -m "message": This command is used to save your staged changes to the repository. The -m flag is used to specify a commit message, which is a brief description of the changes you are committing.

    git push: This command is used to upload your local commits to a remote repository. It allows you to share your changes with other people.

    git pull: This command is used to download the latest changes from a remote repository and merge them into your local copy. It allows you to stay up to date with the latest changes made by other people.

    git status: This command is used to view the current status of your repository. It shows you which files have been modified, staged, or are untracked.

    git log: This command is used to view the commit history of your repository. It shows you a list of all the commits that have been made, along with the commit message and author.

    I hope this cheat sheet helps you get started with Git! If you have any questions or need more information, there are many resources available online that can help you learn more about Git and version control.
